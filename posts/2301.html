对华为5月19日发布的新“笔记本”的初步分析

简单的【云谈论】（未接触实机的空谈）一下华为的新“笔记本”：

形态非常好非常有趣，不过有以下几点我比较在意：

  1. 虚拟键盘的手感问题。利用马达模拟触控板的手感苹果已经做的很成熟了，华为也在它的高端笔记本上做过这个技术。即便如此，让它来模拟键盘触感恐怕还是有一定难度。考虑到存在“键程”的概念，真实的键程和有触感但0键程的键盘，恐怕还是有区别的。更重要的是，使用时敲击键盘的速度和频率显著快于触控板的按压速度，这对触感的模拟将是一个很大的挑战。

  2. 华为提供了一个可以磁吸在主机（我尚且不愿意称之为电脑）背部的无线键盘。这允许用户在有需要时，将主机完全展开作为大屏，并配合键盘使用。我很喜欢这个使用模式，但键盘带来的整机重量还有待观察。

  3. 风扇。部分自媒体将主机内置风扇作为亮点，在我看来这是败笔。作为一个ARM架构的，无游戏等中重负载场景的设备，风扇显得毫无必要，反而平添设计成本、损坏几率、维护和维修成本以及可能的运行时噪音。我完全看不懂这一设计。

关于系统，我目前的看法是这些：

华为不选择基于debain是一个很有勇气的行为。当然，让自家的桌面系统和移动端系统尽可能统一，让开发者可以用最小成本移植移动端应用，并利用自己的品牌号召力扩大新生态，并非不可行的做法。但从整个行业的视角来看，不选择即将趋于成熟的国内linux应用环境而选择自立门户，是在分裂尚未成熟的国内应用市场，给开发者额外增添维护成本（从win + macos + linux + android + ios + harmony移动 到 win + macos + linux + android + ios + harmony移动 + harmony桌面）。除非华为将harmony移动和桌面的开发环境统一到极致，乃至可以共享ui甚至共享安装包（这很难，因为很多应用的桌面端UI和移动端差距很大，如果真的直接0成本迁移，把移动端app在桌面运行，用户使用体验将极差），否则对开发者而言开发和维护成本只增不减。这是有利于自身（还不一定呢）但有损全行业的行为。

Windows是落后的操作系统。如果中国消费市场的Windows占有率能跌到60%以下，我将提前宣布中国是世界上最先进的国家。在我的预想里，这种淘汰的方式是换用Linux发行版。国产操作系统在这方面已经颇有建树了。在近期微信Linux版出来以后，国人使用Linux系统的最大障碍已经基本解除。这个技术先进、功能丰富、生态完整的操作系统已经有望走入大众视线了。原本期待华为在这方面进行进一步的助推，结果是华为决定自立门户。观察了华为的说法，它似乎强调【华为系多设备间流转】以及【AI集成】。尤其是为了做到低一点，决定自立门户也是合理的。不过，从淘汰Windows的宏观层面来讲，这一决定依然令我比较失望。

你的下一部电脑还得是电脑。电脑作为电脑，是有它独特的使用模式的。我总结下来，两个使用场景最为核心：【文件管理】和【网页浏览】。对于一个系统是不是“电脑”，我们可以主要衡量这两个标准：【文件管理（包括内部的文件系统以及外部文件浏览器的交互设计）是否通用好用】以及【网络浏览器是否强大】。别的电脑的特征，什么应用侧载，什么带终端，什么键盘快捷键都是次要的，文件管理和网页浏览是退无可退，是电脑使用的根基，是必须要做好的两大功能。很多人在意的生态事实上也是次要的。现在很多应用含有功能完整的网页版，只要有浏览器，很多任务可以借助浏览器完成。而运行它们的前提是具备一个足够好的浏览器。

为了让这个初步分析不要过于“云”，我补了补课，看了华为对自家系统的宣传。华为强调【华为系多设备间流转】以及【AI集成】，似乎有“重塑个人电脑使用范式”的野心。我们先抛开是“你教用户怎么用电脑还是用户教你怎么做电脑”的问题，就算这个新范式真的搞成了，你真的能革掉【文件管理】和【网页浏览】这两大基础功能的命吗？难。

因此，后续如果接触实机，最需要审视的，就是它的系统文件管理器和网页浏览器。

从【我个人】的使用情况出发聊聊生态。在我看来，生态不是说把那几个常用app都具备就行了的。一个系统的生态在于【其“工具类”应用】的丰富程度，在于我是否能利用工具代替重复性活动并优化自己的工作流。
